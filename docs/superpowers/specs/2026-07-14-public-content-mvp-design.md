# KOTONOHA 公开内容源 MVP 设计

**日期：** 2026-07-14
**状态：** 已获方向批准，等待书面规格复核
**目标：** 停用 DK 与帝京 PDF OCR，利用许可明确的公开词汇和语法资料，尽快交付一个可部署到 Vercel 的非商业日语学习网站。

## 1. 背景与决策

当前项目已经完成 Next.js 应用骨架、确定性学习规则、Neon 数据结构和 Better Auth 集成。原计划的下一阶段依赖 DK 词典和帝京语法 PDF 的 OCR，但真实运行表明 PDF 版式、字符识别和字段对齐会带来大量人工复核，而且原书内容的再发布许可也不够明确。

首版改为公开数据源方案：

- 不再 OCR、解析或发布两份 PDF 的派生内容。
- 不保留 DK 原书目录、页码或帝京 1–228 编号。
- 复用已完成的应用、认证、数据库和学习进度基础。
- 先发布数量受控、来源可靠的内容快照，再通过同一导入接口逐步扩充。

## 2. 首版范围

### 2.1 必须完成

- 首页：今日一词、词汇与语法入口、登录状态和继续学习入口。
- 词汇目录：常用词、名词、动词、形容词及数据中确有依据的主题标签。
- 词汇学习：列表、详情、翻卡和“忘记 / 模糊 / 认识”复习评分。
- 语法目录：书写系统、基础语法、动词、助词、形容词和常用表达。
- 语法详情：表达、中文说明、接续、日文例句、中文译文和来源归属。
- 全站搜索：至少支持日文表记、假名、罗马字和中文释义。
- 游客体验：无需登录即可浏览、搜索和完成一次本地学习流程。
- 登录体验：收藏、评分和下次复习时间同步到 Neon。
- 来源说明：展示内容来源、许可证、快照日期和修改说明。
- Vercel 生产构建、响应式页面和关键路径自动化测试通过。

### 2.2 首版暂缓

- 完整导入全部 JMdict 或 Kaikki 数据。
- 声称为官方 JLPT N1–N5 词表的分级。公开来源无法证明时不展示该标签。
- 音频、手写识别、管理员后台、社交、排行榜和付费功能。
- 广告、赞助、订阅或其他商业化功能。

## 3. 内容来源与许可边界

### 3.1 词汇

主数据使用 EDRDG 的 JMdict。首版从带有常用优先级标记的记录中构建约 2,000 条候选词，保留日文表记、读音、词性、常用标记和英文释义。JMdict 及派生数据遵循 CC BY-SA 4.0，并在站内来源页提供署名和许可证链接。

中文释义使用 Kaikki 提供的中文维基词典结构化 JSONL。仅处理 `lang_code = ja` 的日语词条，并按规范化日文表记和读音匹配 JMdict。中文维基词典派生字段遵循其相应的 CC BY-SA 许可和署名要求。

首版发布规则：

- 必须有日文表记或纯假名表记、有效读音、词性和至少一个可靠中文释义。
- 匹配不唯一、释义为空或语言字段异常的记录进入拒绝清单，不进入公开数据库。
- 罗马字由本地确定性转换生成，并标记为应用派生字段。
- 内容分类仅使用词性和源数据确有的主题字段，不根据直觉编造主题。

### 3.2 语法

语法课程以 Tae Kim 的 Japanese Grammar Guide 为结构和参考来源，首版选择约 30–50 个基础学习单元。原始内容采用 CC BY-NC-SA 3.0，因此：

- KOTONOHA 首版必须保持非商业，不放广告、不收费、不通过内容获利。
- 中文说明以面向中文学习者的重新组织和改写呈现，并标记为派生内容。
- 来源页显示作者、原始页面、许可证以及应用做出的修改。
- 语法内容数据包以兼容的 CC BY-NC-SA 3.0 条款发布；应用代码与内容数据分开标识许可。
- 若未来商业化，必须先整体关闭该内容源，并替换为可商用或原创内容。

首版不直接复制无明确许可的第三方中文译文。日文例句沿用合法源内容时保留归属；新增例句必须由项目原创并在数据中标记来源类型。

### 3.3 来源清单

每次导入生成机器可读清单，至少记录：

- `source_key`、来源名称和公开 URL；
- 许可证名称和许可证 URL；
- 上游快照日期、下载时间与文件 SHA-256；
- 导入器版本、候选数、发布数、拒绝数和拒绝原因统计；
- 是否启用以及最后一次成功导入时间。

线上查询始终过滤 `content_sources.enabled = true`。这样可以在许可、质量或来源可用性发生问题时整体下架一个来源，而无需删除用户进度。

## 4. 系统架构

### 4.1 应用层

继续使用现有 Next.js App Router、React、TypeScript 和 Tailwind 应用。页面只通过稳定的内容查询接口读取词汇、语法、搜索结果和今日内容，不直接理解上游数据格式。

主要模块：

- `content queries`：根据启用来源、分类、搜索词和分页条件查询内容。
- `study rules`：复用现有确定性评分与下次复习日期逻辑。
- `auth and progress`：复用 Better Auth 与 Neon 用户进度存储。
- `source attribution`：提供网页来源说明和单条内容的来源标识。

### 4.2 内容导入层

导入工具与 Web 请求完全隔离，分为四个边界清晰的阶段：

1. 下载或读取固定版本的公开数据文件并校验哈希。
2. 将不同来源转换成内部候选模型。
3. 规范化、匹配、验证并输出发布记录与拒绝报告。
4. 以事务方式写入 Neon；失败时保持上一批已发布内容不变。

导入器必须支持离线测试夹具。测试和普通页面构建不得自动下载数百 MB 的上游数据。

### 4.3 数据模型

复用现有 `content_sources`、内容目录、词汇、语法和用户进度表，并按需要增加以下字段：

- 内容级 `source_id`、`source_entry_key` 和 `license_key`；
- `published`、`review_status` 和 `content_version`；
- 中文释义与例句的来源类型；
- 导入批次、原始快照哈希和拒绝原因。

上游记录使用稳定自然键进行 upsert。内容更新不能改变用户复习记录关联的内部 ID；来源被关闭时用户进度保留但不出现在学习队列。

## 5. 页面与学习流程

### 5.1 词汇流程

`首页 → 词汇目录 → 分类/列表 → 词汇详情或翻卡 → 评分 → 下一条`

首版目录不再显示 DK 品牌或原书层级。用户看到真实发布数量；数量从数据库查询，不使用估算值。词汇详情优先显示日文、假名、中文释义和词性，罗马字与英文释义作为辅助信息，底部显示来源。

### 5.2 语法流程

`首页 → 语法目录 → 单元 → 语法详情 → 收藏或评分`

目录采用学习顺序而不是帝京编号。每个详情页只讲一个明确语法主题，包含接续、解释和至少一个经过验证的例句。缺少中文说明或例句译文的单元不得发布。

### 5.3 游客与登录用户

游客的临时学习状态保存在浏览器本地，可浏览全部公开内容。尝试跨设备同步或长期收藏时提示登录。登录后只上传用户明确产生的收藏和评分，不自动覆盖服务器上更新的进度。

## 6. 错误处理与数据质量

- 下载失败或哈希不符：终止导入，不修改线上数据。
- 解析失败：记录源文件位置和错误类型，超过阈值时终止整个批次。
- 中日释义匹配不唯一：拒绝该候选，不猜测合并。
- 数据库事务失败：回滚本批次，继续提供上一版本内容。
- 来源被禁用：页面返回正常的“内容暂不可用”状态，不暴露已下架正文。
- 搜索无结果：提供清除筛选和返回目录入口，不生成伪结果。
- 第三方许可发生变化：通过来源开关下架，重新评估后再导入。

拒绝报告必须可以复现并包含聚合统计，但不得把整份受许可约束的上游数据提交进 Git 仓库。

## 7. 测试与验收

### 7.1 内容测试

- 上游解析器使用小型、可提交的许可兼容夹具测试。
- 验证日文、读音、中文释义、词性、稳定键和来源字段完整。
- 验证重复候选、歧义匹配和空字段会进入拒绝清单。
- 同一快照重复导入必须幂等。
- 来源关闭后，公开查询和学习队列均不返回该来源内容。

### 7.2 应用测试

- 单元测试覆盖学习评分、搜索规范化、罗马字转换和内容过滤。
- 组件测试覆盖首页、词汇卡片、语法详情、空状态和来源说明。
- 端到端测试覆盖游客试学、登录收藏、评分同步、搜索与移动端导航。
- 生产构建不得依赖网络下载，且不得包含 PDF、OCR 缓存或上游完整原始文件。

### 7.3 上线验收

首版满足以下条件即可部署：

- 至少 500 条真实、带中文释义的公开词汇；目标为约 2,000 条。
- 至少 30 个完整语法单元。
- 游客可以完成从首页到一次评分的闭环。
- 登录用户刷新页面后仍能看到收藏和复习进度。
- 来源、署名、许可证和非商业声明可从全站页脚访问。
- `lint`、类型检查、单元测试、端到端冒烟测试和生产构建全部通过。
- 在 360、768、1024 和 1440 像素宽度下没有阻断使用的问题。

## 8. 部署与后续扩展

首版部署目标保持为 Vercel + Neon。内容数据在部署前导入 Neon，Vercel 构建只编译应用。环境变量仅保存数据库和认证密钥；内容来源清单与许可元数据可公开读取。

上线后按以下顺序扩展：

1. 扩大 JMdict 与 Kaikki 匹配覆盖率并加入人工复核队列。
2. 增加更多 Tae Kim 语法单元。
3. 接入许可明确的例句来源。
4. 在找到可证明来源的开放分级表后，再增加 JLPT 或其他能力等级筛选。

任何扩展都必须继续遵守“不编造内容、来源可追踪、可整体下架、导入可复现”的原则。
