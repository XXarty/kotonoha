# KOTONOHA 相邻内容导航设计

## 目标

让学习者在单词和语法详情页中自然地继续阅读，不必返回目录，同时保持当前学习分类和难度上下文。

## 已确认顺序

### 单词

- 相邻项限定在当前条目的“相同词性分类 + 相同层级”中。
- 词性分类使用现有 `category`：`nouns`、`verbs`、`adjectives`、`other`。
- 层级使用现有 `tier`：`core` 或 `extended`。
- 顺序复用对应列表当前的稳定数据顺序，不建立新的频率或 JLPT 排名。
- 不跨词性、不跨层级循环。

### 语法

- 相邻项限定在当前条目的同一 `path` 中。
- 使用现有 `display_order`，并沿用仓库中 ID 作为同序号时的稳定次级排序。
- 不跨基础、核心、常用表达、进阶路径循环。

## 数据接口

内容仓库新增两个只读查询：

- `getVocabularyNeighbors(id: string): { previous: VocabularyEntry | null; next: VocabularyEntry | null }`
- `getGrammarNeighbors(slug: string): { previous: GrammarEntry | null; next: GrammarEntry | null }`

未知 ID/slug 返回 `{ previous: null, next: null }`，详情页原有 404 逻辑保持不变。

## 页面表现

- 导航位于主要释义/例句内容之后、学习评分之前。
- 上一个按钮显示“上一个”及相邻条目的日文主文本。
- 下一个按钮显示“下一个”及相邻条目的日文主文本。
- 单词链接使用稳定内容 ID 路由；语法链接使用 slug 路由。
- 当前分组的第一个条目不渲染“上一个”，最后一个条目不渲染“下一个”。
- 不循环回到分组另一端。
- 桌面端左右对齐；手机端纵向排列，触摸目标至少 44px。
- 使用语义化 `<nav aria-label="相邻内容">`，链接保持清晰的键盘焦点样式。

## 与动态背景的关系

相邻导航与已确认的和纸动态背景一起实施。背景层不拦截链接点击，减少动态模式下仍保持静止。

## 验收标准

- 单词不会跨词性或层级跳转。
- 语法不会跨学习路径跳转。
- 分组首尾边界正确，无循环和失效链接。
- 桌面与手机端按钮清晰、可点击、无横向溢出。
- 内容仓库测试、详情页测试、类型检查、构建、E2E 与浏览器视觉验收通过。
