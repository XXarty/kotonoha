# KOTONOHA 产品需求文档

**版本：** MVP 2.0
**日期：** 2026-07-14
**产品类型：** 面向中文母语者的日语词汇与语法学习网站
**部署目标：** GitHub + Vercel；Neon 仅用于账户与学习进度
**商业状态：** 非商业

## 1. 产品概述

KOTONOHA 将许可明确的公开日语资料整理成可浏览、可搜索、可记忆、可复习的学习路径。公开内容由离线工具从 JMdict、Kaikki 中文维基词典与 Tae Kim 语法指南构建为小型 JSON 快照，随代码提交到 GitHub，并由 Vercel 直接打包。

项目不再 OCR、解析或发布 DK 与帝京 PDF 内容，也不保留其目录、页码或编号。Neon 不保存公开词汇和语法正文，只保存 Better Auth 数据、收藏、评分、复习计划与学习会话。

## 2. 产品目标

1. 用户在 30 秒内进入词汇、五十音或语法学习。
2. 游客无需登录即可浏览、搜索和完成一次本地评分。
3. 登录用户能够跨设备同步收藏和复习进度。
4. 每条公开内容都能追溯到来源、许可证与内容版本。
5. 新内容通过离线构建和提交 JSON 扩充，不改写页面结构。
6. 公共页面在没有 Neon 环境变量时仍可构建和访问。

## 3. 目标用户

- 中文母语的日语初学者。
- 想按常用词和词性积累词汇的人。
- 已学习基础语法、需要系统复习的人。
- 希望用电脑与手机保持同一学习进度的人。

核心任务包括浏览目录、查词、理解语法、学习五十音、收藏内容和完成每日复习。

## 4. 首发范围

### 必须上线

- 首页：今日一词、词汇/语法双入口、五十音和继续学习。
- 词汇：常用名词、动词、形容词和其他可靠分类。
- 词汇详情：日文、假名、罗马字、词性、中英文释义和来源。
- 五十音：46 个基础平假名/片假名对。
- 语法：至少 30 个基础单元，包含接续、中文解释和双语例句。
- 搜索：支持日文、假名、罗马字和中文释义。
- 学习：忘记、模糊、认识三档评分。
- 账户：邮箱密码登录、收藏、复习进度与跨设备同步。
- 来源与许可：来源、版本、哈希、许可证、修改说明与非商业声明。

### 首发暂缓

- 完整 JMdict/Kaikki 内容。
- 无法证明来源的 JLPT N1-N5 词表或官方词量。
- 音频、手写识别、管理员后台、社交和排行榜。
- 广告、付费、订阅、赞助或其他商业化功能。

## 5. 信息架构与路由

| 路由 | 用途 |
| --- | --- |
| `/` | 首页与今日一词 |
| `/vocabulary` | 词汇分类目录 |
| `/vocabulary/[category]` | 分类词汇列表 |
| `/vocabulary/entry/[id]` | 词汇详情与学习评分 |
| `/kana` | 五十音表 |
| `/grammar` | 语法分类目录 |
| `/grammar/[category]` | 分类语法列表 |
| `/grammar/entry/[slug]` | 语法详情与学习评分 |
| `/search` | 静态内容搜索 |
| `/review` | 登录用户的到期复习队列 |
| `/sources` | 来源、许可和版本说明 |
| `/sign-in` | 登录与注册 |
| `/profile` | 账户、收藏和学习摘要 |

所有内容 ID 使用稳定字符串，例如 `vocabulary:jmdict:1000001`、`grammar:tae-kim:te-form` 和 `kana:gojuon:a`。

## 6. 页面需求

### 首页

- 显示从启用来源中确定性选出的今日一词。
- 同一香港自然日保持不变；无候选时显示进入词汇目录的正常空状态。
- 提供词汇、语法、五十音和复习入口。
- 不读取 Neon 公开内容表。

### 词汇

- 分类数量从静态内容索引计算，不显示估算值。
- 列表支持快速扫描和键盘访问。
- 详情优先显示日文、假名、中文释义和词性；英文与罗马字作为辅助。
- 详情页靠近正文显示来源与许可证。

### 五十音

- 显示 46 个基础平假名、片假名和罗马字配对。
- 按五十音行组织，移动端不横向溢出。
- 首发不包含浊音、半浊音和拗音扩展。

### 语法

- 目录按学习顺序和主题组织，不显示帝京编号。
- 每个详情只讲一个明确主题，包含表达、接续、中文说明和至少一个双语例句。
- 缺少中文说明、例句译文或来源的条目不得进入发布快照。

### 搜索与复习

- 搜索在内存中对最多约 2,000 个词和 30 个语法单元执行 NFKC 规范化匹配。
- 空搜索不产生伪结果；无结果时提供清除输入与返回目录入口。
- 复习页只从 Neon 读取当前用户的到期 ID，再由静态仓库补全内容。
- 已禁用或已移除的内容不展示，但不删除用户进度。

## 7. 学习进度

- 游客评分保存在浏览器 IndexedDB，不写入 Neon。
- 登录用户的评分由服务端重新验证身份、内容 ID 和复习规则后写入 Neon。
- 客户端不得提交用户 ID、下次复习时间、连续正确次数或最终状态。
- 收藏与进度使用 `(user_id, kind, item_id)` 稳定身份去重。
- 登录或网络失败不得阻断公开内容浏览。

## 8. 公开内容与来源

### 词汇

- JMdict Simplified common 数据提供日文表记、读音、词性和英文释义。
- Kaikki 中文维基词典提供可靠中文释义。
- 只接受唯一、可解释的匹配；空释义和歧义同形词进入聚合拒绝统计。
- 罗马字由确定性本地转换生成。

### 语法

- 课程结构参考 Tae Kim Japanese Grammar Guide。
- 中文说明为面向中文学习者的重新组织和改写，不复制许可不明的第三方中文译文。
- 内容遵循 CC BY-NC-SA 3.0；站点必须保持非商业。
- 未来商业化前必须整体关闭并替换该来源。

### 五十音

- 46 个基础配对为项目整理的事实数据，以 CC0 1.0 发布。

## 9. 静态内容构建

离线构建流程：

1. 从允许的 HTTPS 域名下载固定版本上游文件并计算 SHA-256。
2. 将来源转换为内部候选模型。
3. 规范化、匹配、验证并记录聚合拒绝原因。
4. 在临时目录生成规范 JSON，全部复核通过后替换上一版文件。
5. 提交 `webapp/src/content/generated/` 到 GitHub，Vercel 构建直接导入。

Git 只包含 `sources.json`、`vocabulary.json`、`grammar.json`、`kana.json`、`manifest.json`、验证结果和许可说明。完整上游文件、PDF、缓存、构建目录、密钥和数据库导出不得提交。

来源清单包含来源 URL、许可证、快照日期、上游哈希、启用状态、候选/发布/拒绝数量和生成器版本。将 `enabled` 改为 `false` 并重新部署即可下架一个来源。

## 10. 数据与技术架构

- Next.js App Router + React + TypeScript + Tailwind CSS。
- 公共内容通过 TypeScript 静态仓库读取 JSON，不查询 Neon。
- Better Auth 和 Neon PostgreSQL 只保存账户及用户数据。
- Drizzle 管理用户数据迁移；`user_item_progress.item_id` 与 `favorites.item_id` 为文本。
- React Server Components 默认；本地评分等浏览器交互使用 Client Components。
- Vercel 构建不得访问上游词典，也不得要求 `DATABASE_URL` 才能生成公开页面。

## 11. 视觉设计

采用“现代日语词汇标本室”方向：冷调和纸白背景、排版、细线和索引结构，不使用樱花、鸟居或动漫陈词滥调。

| Token | 色值 | 用途 |
| --- | --- | --- |
| Paper | `#F5F6F2` | 主背景 |
| Ink | `#20282B` | 主文本 |
| Mineral blue | `#315F80` | 链接与当前状态 |
| Celadon | `#CAD5C8` | 轻分组 |
| Lacquer red | `#A43630` | 少量强调 |
| Mist | `#DDE3E2` | 分隔线与禁用状态 |

展示与日文标题使用日文衬线字体，中文和界面使用清晰无衬线字体。“词汇标本条”展示大号日文、假名、释义、来源和细长索引。动效必须短促并遵守 `prefers-reduced-motion`。

## 12. 无障碍与响应式

- 所有操作可由键盘完成并有可见焦点。
- 正文对比度满足 WCAG AA。
- 360px 宽度不出现页面级横向滚动。
- 触控目标不小于 44x44px。
- 语义列表、标题和状态提示可被读屏读取。
- 装饰纹理不进入读屏顺序。

## 13. 异常与降级

- 上游下载、哈希或验证失败：构建退出，不替换上一版快照。
- Neon 不可用：公开浏览和搜索继续工作；登录与云端同步显示明确提示。
- 来源禁用：重新部署后目录、详情、搜索、今日一词和复习队列均不显示该来源。
- 今日一词无候选：显示目录入口，不生成虚假词条。
- 搜索无结果：显示正常空状态，不调用外部搜索或生成内容。

## 14. 测试与上线验收

- 至少 500 条真实带中文释义词汇，目标约 2,000 条。
- 恰好 30 个完整基础语法单元与 46 个基础假名对。
- 同一快照重复构建生成字节一致的 JSON。
- 静态文件哈希、清单数量和验证结果一致。
- 来源关闭测试覆盖目录、详情、搜索、今日一词与复习补全。
- 游客能完成首页到一次评分的闭环。
- 登录用户刷新后仍能看到收藏和复习进度。
- `npm test`、lint、类型检查、生产构建和端到端冒烟测试通过。
- 没有 Neon 环境变量时，公开页面仍能完成生产构建。
- 360、768、1024、1440 像素宽度无阻断使用问题。

## 15. 发布流程

1. 生成并验证静态内容快照。
2. 运行 Python、TypeScript、组件和端到端测试。
3. 提交不含上游原文件、PDF 和密钥的代码与生成内容到 GitHub。
4. 从 GitHub 部署到 Vercel；公开浏览无需 Neon。
5. 需要账户同步时，在 Neon 应用用户数据迁移并配置 Better Auth 环境变量。
6. 在线验证首页、词汇、五十音、语法、搜索、来源说明和可选登录进度。

## 16. 成功指标

- 首次访问到开始学习少于 60 秒。
- 目录到具体学习内容不超过 3 次点击。
- 公开内容页面不受 Neon 故障影响。
- 登录用户跨设备恢复进度准确。
- 来源下架在一次 GitHub 提交与 Vercel 重新部署后生效。
